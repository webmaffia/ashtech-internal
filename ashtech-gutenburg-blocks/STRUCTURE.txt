AFTER DEPLOYMENT:
=================

Production server only needs:
- Plugin PHP file
- build/ folder
- assets/ folder
- block.json files

You DON'T need to upload:
- src/ folder (source code)
- node_modules/
- package.json
- webpack.config.js

(Though keeping them allows future updates)


BLOCK LOADING ORDER:
====================

WordPress loads blocks in this sequence:

1. ashtech-gutenberg-blocks.php registers all blocks
2. Each block's build/blocks/[name]/index.js loads
3. Block appears in "Ashtech Blocks" category
4. User adds block to page
5. edit.js renders in editor
6. save.js renders on frontend
7. styles from build/blocks/[name]/style-index.css apply
8. Global assets from assets/ folder load


CUSTOMIZATION POINTS:
=====================

Easy:
- Block content (via WordPress editor)
- Block attributes (in Inspector Controls)
- Asset files (images, CSS)

Medium:
- Block edit.js (editor interface)
- Block save.js (frontend output)
- Block styles (SCSS files)

Advanced:
- New blocks (create new folder in src/blocks/)
- New attributes (modify block.json)
- PHP functionality (modify main plugin file)


BACKUP RECOMMENDATIONS:
=======================

Essential files to backup:
- src/ folder (all source code)
- assets/ folder (all media files)
- Documentation files
- package.json
- webpack.config.js
- Main PHP file

Can regenerate:
- build/ folder (via npm run build)
- node_modules/ (via npm install)
